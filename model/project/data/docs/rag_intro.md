# RAG 개요

Retrieval-Augmented Generation, 줄여서 RAG는 외부 지식 문서를 검색한 뒤 검색된 문맥을 언어 모델의 입력으로 제공하여 답변을 생성하는 방법이다.

일반적인 RAG 파이프라인은 문서 수집, 청크 분할, 임베딩 생성, 벡터 검색, 프롬프트 구성, 답변 생성 단계로 구성된다. 사용자의 질문이 들어오면 질문도 같은 임베딩 모델로 벡터화하고, 문서 청크 벡터와의 유사도를 계산해 관련 문맥을 찾는다.

RAG의 장점은 모델 파라미터 안에 없는 최신 정보나 사내 문서를 답변에 활용할 수 있다는 점이다. 또한 답변 근거가 되는 문서를 함께 보여주면 사용자가 결과를 검증하기 쉬워진다.
